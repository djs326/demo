## 开发规范

### 后端开发规范

#### 1. 包命名规范

- 控制器: `com.example.xxx.controller`
- 服务接口: `com.example.xxx.service`
- 服务实现: `com.example.xxx.service.impl`
- 数据访问: `com.example.xxx.mapper`
- 实体类: `com.example.xxx.entity` 或 `com.example.xxx.model`
- DTO: `com.example.xxx.dto`
- VO: `com.example.xxx.vo`
- 配置类: `com.example.xxx.config`
- 工具类: `com.example.xxx.utils`
- 异常: `com.example.xxx.exception`

#### 2. 类命名规范

- 控制器: 以 `Controller` 结尾，如 `UserController`
- 服务接口: 以 `Service` 结尾，如 `UserService`
- 服务实现: 以 `ServiceImpl` 结尾，如 `UserServiceImpl`
- Mapper: 以 `Mapper` 结尾，如 `UserMapper`
- 实体类: 使用名词，如 `User`
- 配置类: 以 `Config` 结尾，如 `SwaggerConfig`
- 工具类: 以 `Utils` 或 `Helper` 结尾，如 `DateUtils`

#### 3. 方法命名规范

- 查询单个对象: `getXxx` 或 `findXxx`
- 查询列表: `listXxx` 或 `findXxxList`
- 分页查询: `pageXxx`
- 新增: `addXxx` 或 `saveXxx` 或 `createXxx`
- 修改: `updateXxx` 或 `modifyXxx`
- 删除: `deleteXxx` 或 `removeXxx`
- 判断存在: `existsXxx`
- 统计: `countXxx`

#### 4. RESTful API 规范

| 操作 | HTTP 方法 | 路径示例 |
|------|-----------|----------|
| 查询列表 | GET | `/api/users` |
| 查询单个 | GET | `/api/users/{id}` |
| 新增 | POST | `/api/users` |
| 修改 | PUT | `/api/users/{id}` |
| 删除 | DELETE | `/api/users/{id}` |

### 前端开发规范

#### 1. 组件命名

- 组件文件名: PascalCase，如 `UserList.vue`
- 组件注册名: PascalCase
- 组件使用: kebab-case 或 PascalCase

#### 2. 文件命名

- 页面组件: PascalCase，如 `Home.vue`
- 工具函数: camelCase，如 `dateUtils.js`
- 常量: UPPER_SNAKE_CASE，如 `API_BASE_URL`

#### 3. 代码风格

- 使用 Vue 3 Composition API
- 使用 `<script setup>` 语法
- 组件内使用 TypeScript（如已配置）
- 统一使用 2 空格缩进

## 添加新模块

### 后端添加新模块

#### 1. 创建模块目录

在 `back/modules/` 下创建新模块目录，例如 `order`：

```bash
cd back/modules
mkdir -p order/src/main/java/com/example/order
mkdir -p order/src/main/resources/mapper
mkdir -p order/src/test/java/com/example/order
```

#### 2. 创建 pom.xml

```xml
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <parent>
        <artifactId>modules</artifactId>
        <groupId>com.example</groupId>
        <version>1.0.0</version>
    </parent>
    <modelVersion>4.0.0</modelVersion>

    <artifactId>order</artifactId>
    <name>order</name>
    <description>Order Module</description>

    <dependencies>
        <dependency>
            <groupId>com.example</groupId>
            <artifactId>common</artifactId>
            <version>1.0.0</version>
        </dependency>
    </dependencies>
</project>
```

#### 3. 在父模块中注册

编辑 `back/modules/pom.xml`，添加新模块：

```xml
<modules>
    <module>system</module>
    <module>order</module>
</modules>
```

#### 4. 创建实体类

```java
package com.example.order.entity;

import com.baomidou.mybatisplus.annotation.IdType;
import com.baomidou.mybatisplus.annotation.TableId;
import com.baomidou.mybatisplus.annotation.TableName;
import lombok.Data;

import java.time.LocalDateTime;

@Data
@TableName("t_order")
public class Order {
    @TableId(type = IdType.AUTO)
    private Long id;
    private String orderNo;
    private Long userId;
    private Integer status;
    private LocalDateTime createTime;
    private LocalDateTime updateTime;
}
```

#### 5. 创建 Mapper

```java
package com.example.order.mapper;

import com.baomidou.mybatisplus.core.mapper.BaseMapper;
import com.example.order.entity.Order;
import org.apache.ibatis.annotations.Mapper;

@Mapper
public interface OrderMapper extends BaseMapper<Order> {
}
```

#### 6. 创建 Service

```java
package com.example.order.service;

import com.baomidou.mybatisplus.extension.service.IService;
import com.example.order.entity.Order;

public interface OrderService extends IService<Order> {
}
```

```java
package com.example.order.service.impl;

import com.baomidou.mybatisplus.extension.service.impl.ServiceImpl;
import com.example.order.entity.Order;
import com.example.order.mapper.OrderMapper;
import com.example.order.service.OrderService;
import org.springframework.stereotype.Service;

@Service
public class OrderServiceImpl extends ServiceImpl<OrderMapper, Order> implements OrderService {
}
```

#### 7. 创建 Controller

```java
package com.example.order.controller;

import com.example.common.model.Result;
import com.example.order.entity.Order;
import com.example.order.service.OrderService;
import io.swagger.v3.oas.annotations.Operation;
import io.swagger.v3.oas.annotations.tags.Tag;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.web.bind.annotation.*;

@RestController
@RequestMapping("/api/orders")
@Tag(name = "订单管理", description = "订单相关接口")
public class OrderController {

    @Autowired
    private OrderService orderService;

    @GetMapping("/{id}")
    @Operation(summary = "获取订单详情")
    public Result<Order> getById(@PathVariable Long id) {
        return Result.success(orderService.getById(id));
    }

    @PostMapping
    @Operation(summary = "创建订单")
    public Result<Boolean> create(@RequestBody Order order) {
        return Result.success(orderService.save(order));
    }
}
```

```vue
<template>

<script setup>
</script>

<style scoped>
</style>
```

编辑 `front-admin/src/router/index.js`，添加新路由：

```javascript
import OrderList from '../views/OrderList.vue'

    path: '/orders',

    component: OrderList
  }
]
```

## 代码规范

### Git 提交规范

提交信息格式：

```
<type>(<scope>): <subject>

<body>

<footer>
```

Type 类型：

- `feat`: 新功能
- `fix`: 修复 bug
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 重构
- `perf`: 性能优化
- `test`: 测试相关
- `chore`: 构建/工具相关

示例：

```
feat(user): 添加用户分页查询功能

- 实现分页查询接口
- 添加参数校验

Closes #123
```

### 代码注释

- 类注释：说明类的职责
- 方法注释：说明方法的功能、参数、返回值
- 复杂逻辑：添加行内注释说明

### 异常处理

- 使用统一的异常处理机制
- 业务异常使用 `BusinessException`
- 使用 `ErrorCode` 定义错误码
