# 部署文档

本项目支持 Docker 和 Kubernetes 两种部署方式，以下是详细的部署步骤。

## 目录

- [Docker 部署](#docker-部署)
- [Kubernetes 部署](#kubernetes-部署)

---

## Docker 部署

### 前置要求

- Docker 20.10+
- Docker Compose 2.0+

### 部署步骤

#### 1. 克隆项目

```bash
git clone <repository-url>
cd SpringBoot3-Vue3
```

#### 2. 使用 Docker Compose 启动

项目根目录已提供 `docker-compose.yml` 文件，可直接使用：

```bash
docker-compose up -d
```

#### 3. 验证部署

- 后端服务: http://localhost:8080
- 管理前端: http://localhost:8081
- 用户前端: http://localhost:8082
- API 文档 (Knife4j): http://localhost:8080/doc.html

#### 4. 停止服务

```bash
docker-compose down
```

### 自定义配置

如需修改配置，可编辑 `docker-compose.yml` 文件中的环境变量或挂载配置文件。

## Kubernetes 部署

### 前置要求

- Kubernetes 1.20+
- kubectl 已配置
- 可选: Helm 3.0+

### 部署步骤

#### 1. 克隆项目

```bash
git clone <repository-url>
cd SpringBoot3-Vue3
```

#### 2. 创建命名空间

```bash
kubectl apply -f k8s/01-namespace.yaml
```

#### 3. 部署配置文件

```bash
kubectl apply -f k8s/02-configmap.yaml
```

#### 4. 部署密钥 (可选)

```bash
kubectl apply -f k8s/03-secret.yaml
```

注意: 请根据实际情况修改 Secret 中的敏感信息。

#### 5. 部署后端服务

```bash
kubectl apply -f k8s/04-backend-deployment.yaml
```

#### 6. 部署前端服务

```bash
kubectl apply -f k8s/05-frontend-deployments.yaml
```

#### 7. 部署 Service

```bash
kubectl apply -f k8s/06-services.yaml
```

#### 8. 部署 Ingress (可选)

如果需要外部访问，可部署 Ingress：

```bash
kubectl apply -f k8s/07-ingress.yaml
```

#### 9. 部署 HPA (可选)

如需自动扩缩容，可部署 HPA：

```bash
kubectl apply -f k8s/08-hpa.yaml
```

#### 10. 验证部署

查看 Pod 状态：

```bash
kubectl get pods -n springboot-vue3
```

查看 Service 状态：

```bash
kubectl get svc -n springboot-vue3
```

### 访问服务

#### 方式一: Port-Forward

```bash
# 后端
kubectl port-forward svc/backend-service 8080:8080 -n springboot-vue3

# 管理前端
kubectl port-forward svc/front-admin-service 8081:80 -n springboot-vue3

# 用户前端
kubectl port-forward svc/front-user-service 8082:80 -n springboot-vue3
```

#### 方式二: 通过 Ingress 访问

如果已部署 Ingress，可通过配置的域名访问。

### 清理部署

如需删除所有部署资源：

```bash
kubectl delete namespace springboot-vue3
```

## 常见问题

### Docker 部署常见问题

1. **端口被占用**
   - 修改 `docker-compose.yml` 中的端口映射

2. **容器启动失败**
   - 查看容器日志: `docker logs <container-id>`

### Kubernetes 部署常见问题

1. **Pod 一直处于 Pending 状态**
   - 检查节点资源: `kubectl describe nodes`
   - 检查 PVC 绑定状态

2. **服务无法访问**
   - 检查 Service 选择器是否匹配 Pod 标签
   - 检查网络策略配置
